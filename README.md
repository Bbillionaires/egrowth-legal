# eGrowth Legal — SaaS Platform

Trust, Estate Planning, Nonprofit & For-Profit Registration SaaS with Document Generation, Trustee Services, and Human Handoff Queue.

---

## Stack

| Layer | Tech |
|---|---|
| Framework | Next.js 14 (App Router) |
| Database | Supabase (Postgres + RLS) |
| Auth | Supabase Auth |
| Storage | Supabase Storage |
| Documents | docxtemplater + pizzip |
| Email | Resend |
| Payments | Stripe |
| Hosting | Vercel |
| Styling | Tailwind CSS |

---

## Quick Start

### 1. Create the project

```bash
npx create-next-app@14 egrowth-legal --typescript --tailwind --app --eslint
cd egrowth-legal
```

### 2. Install dependencies

```bash
npm install @supabase/supabase-js @supabase/ssr docxtemplater pizzip file-saver \
  react-hook-form zod @hookform/resolvers clsx tailwind-merge lucide-react \
  date-fns resend stripe @stripe/stripe-js geist
```

### 3. Copy files

Copy all files from this scaffold into your project, preserving the directory structure.

### 4. Environment variables

```bash
cp .env.local.example .env.local
```

Fill in your keys:
- `NEXT_PUBLIC_SUPABASE_URL` — from Supabase project settings
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` — from Supabase project settings → API
- `SUPABASE_SERVICE_ROLE_KEY` — from Supabase project settings → API (keep secret!)
- `RESEND_API_KEY` — from resend.com
- `STRIPE_SECRET_KEY` — from stripe.com dashboard

### 5. Run database schema

Open Supabase → SQL Editor → paste and run `supabase/schema.sql`

### 6. Create storage buckets

The schema.sql creates these, but verify in Supabase → Storage:
- `documents` — generated DOCX files
- `trustee-vault` — client file custody
- `signed-documents` — signed/notarized docs

### 7. Create your master account

In Supabase → Auth → Users → "Invite user" with your email.

Then in SQL Editor:
```sql
UPDATE profiles SET role = 'master' WHERE email = 'your@email.com';
```

### 8. Run dev server

```bash
npm run dev
```

Open http://localhost:3000

## Document Templates

Place your `.docx` template files in `src/templates/`. 

Template variables use `{variable_name}` syntax (docxtemplater).

Available variables:
- `{full_name}`, `{email}`, `{phone}`, `{dob}`, `{state}`, `{date_today}`
- `{trust_name}`, `{trust_type}`, `{successor_trustee_name}`, `{beneficiaries}`
- `{entity_name}`, `{entity_type}`, `{registered_agent}`, `{members}`, `{purpose}`
- `{org_name}`, `{mission_statement}`, `{board_members}`, `{tax_year}`
- `{trustee_company}`, `{trustee_address}`, `{flat_fee}`, `{pct_fee}`

Required template files:
```
src/templates/
  revocable_living_trust.docx
  irrevocable_trust.docx
  land_trust.docx
  pour_over_will.docx
  power_of_attorney.docx
  healthcare_directive.docx
  articles_of_organization.docx
  operating_agreement.docx
  articles_of_incorporation.docx
  bylaws.docx
  trustee_agreement.docx
  file_custody_agreement.docx
  fee_schedule.docx
```

Until you add real templates, the system generates structured placeholder DOCX files automatically.

## Role System

```
Master Account
  └── Creates Admins
      └── Creates Staff
      └── Creates Notaries
          └── All create Client accounts
```

- **Master** — full access, cannot be modified
- **Admin** — manages all platform features, creates staff
- **Staff** — processes queue, updates client records
- **Notary** — marks docs notarized, uploads seal
- **Client** — views own docs, trustee portal

---

## Human Handoff Queue (IP Protection)

Documents are generated by the platform. A staff member then manually logs into state portals and submits.

This prevents:
- IP bans from state websites
- Bot detection
- Rate limiting

Staff workflow:
1. Interview completed → docs auto-queued
2. Staff member assigned in Queue page
3. Staff reviews docs → marks "In Review"
4. Staff manually files with state
5. Enters confirmation number → marks "Submitted" → "Complete"

## Trustee Service

When "eGrowth Co-Trustee" is selected:
- Trustee account created automatically
- Files stored in `trustee-vault` bucket
- Flat fee + percentage tracked in `trustee_ledger`
- Billing managed in Trustee Portal
